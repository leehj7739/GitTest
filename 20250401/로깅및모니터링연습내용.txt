신규 api

====
로그 작성 및, 컨테이너 화면에 표기


docker-compose build --no-cache


=
신규 api로 랜덤 음식 추천 서비스 작성
=> /foodRec 접속시 랜덤 3을 돌려 다음의 api로 리턴
=> /hamburgur, /chicken, /pizza
=> 각 주소 접근시 html파일과 연동하여 이미지 팝업 및, 로그 작성됨


html 파일을 fast api 서버에 업로드하여 해당 api접근시 이미지를 팝업
=> html을 사용하기위해 main.py에 import 추가
=> html 파일을 직접 저장하고 copy를 통해 내 저장소에서 해당 파일을 fastapi 컨테이너의 /app/templetes/some.html 형태로 저장
--> 해당부분에서 볼륨 매칭시 주소 표현에서 에러발생하여 수정함

=> 그라파나를 통해 loki를 연동하여 대시보드에 시각화
=> 모든로그, api별 로그, 일정시간동안 로그를 그래프 시각화 
=> 너무 짧은 시간에 콜할경우 서버 트래픽 급증, 그라파나에서도 정상적으로 로그를 수집하지 못함
=> 수집되는 로그 시간을 적당하게 조절 및, 리프레시 타임도 적당하게 조절
